Conceito Base

Sistemas Distribuídos

Um sistema distribuído é quando um sistema principal faz integração com outros sistemas para aumentar informações ou serviços do qual o sistema principal não possui.

Arquitetura

Cliente-Servidor

Sistema onde um servidor central possui os dados e os clientes buscam esses dados.

Vantagem: É seguro pois as regras de acesso e protocolos de segurança são aplicados centralmente, protegendo as informações de forma eficiente.

Desvantagem: Se o servidor central cair, todos os outros sistemas também caem.

Arquitetura

Peer-to-Peer (P2P)

Dois sistemas compartilham dados entre si, um envia para o outro e vice-versa, sem depender de um servidor central.

Vantagem: Maior escalabilidade, ou seja, a capacidade de manter a eficiência com o aumento de dados ou usuários.

Desvantagem: Alto consumo de recursos computacionais devido ao grande número de requisições.

Arquitetura

Orientado a Serviços

Um sistema oferece serviços para outros sistemas através da rede.

  • Armazenamento em Nuvem
  • Serviços de Inteligência Artificial
  • APIs Especializadas
Alto Desempenho

Supercomputação

A supercomputação é o uso de supercomputadores para a resolução de problemas.. Os supercomputadores combinam milhares de processadores trabalhando paralelamente, o que o torna muito mais rápido, sendo possível resolver problemas complexos que seriam impossíveis de processar em computadores comuns em um tempo razoável.

São comumente utilizados em:

  • Previsões meteorológicas
  • Estudos climáticos
  • Biotecnologia
Execução

Processos e Concorrência

Processo: Unidade de execução que processa uma requisição (um programa em execução).

Concorrência: Ocorre quando dois ou mais processos tentam acessar ou modificar o mesmo recurso ao mesmo tempo. O que pode gerar a condição de corrida.

⚠️ Condição de Corrida: O resultado final depende da ordem de execução. Se 2 processos atualizam algo ao mesmo tempo, um valor pode sobrescrever o outro.

Integração

Comunicação de Processos

Ocorre por meio da troca de mensagens por uma rede de dados. O processo que está enviando empacota a informação em um formato padronizado, e o protocolo de rede garante que esses bits cheguem ao processo que enviará as informações, que então desempacota e processa o conteúdo.

RPC

Remote Procedure Call. Faz a chamada em um computador remoto parecer uma função local para o desenvolvedor.

MOM

Message-Oriented Middleware. Comunicação totalmente baseada em filas de mensagens.

Controle

Sincronização

Garante a ordem dos eventos, coordena o acesso a recursos e garante a consistência de dados. A falta dela gera o problema de inconsistência de estado.

⚠️ Inconsistência de Estado: Sem sincronização, dois processos podem ler que um recurso está disponível ao mesmo tempo (ex: vender o mesmo ingresso duas vezes).

Resiliência

Tolerância a Falhas

Capacidade de continuar operando corretamente, mesmo que componentes (hardware, software, rede) falhem.

O sistema é projetado para detectar o erro, isolá-lo e recuperar-se de forma transparente para o usuário final.

Isso aumenta drasticamente a confiabilidade do sistema durante sua operação, já que a confiabilidade do sistema é a probabilidade desse sistema funcionar sem falhas durante um certo período.

Agrupamento

Cluster vs Grid

Cluster de Computadores

Nós no mesmo espaço físico, na mesma rede local. Rodam o mesmo S.O. e funcionam como um único sistema para realizar uma tarefa. Ele melhora o desempenho das aplicações, visto que ele divide as tarefas grandes em pedaços pequenos e cada nó do cluster processa uma parte simultaneamente.

Grid Computing

É o conjunto de recursos de computação de vários lugares, ele utiliza o poder de processamento de computadores espalhados pelo mundo, via internet. Não exige o mesmo S.O. nem o mesmo hardware. Descentralizado e heterogêneo.
